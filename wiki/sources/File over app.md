# File over app

[[Steph Ango]] presents [[File Over App]] as a philosophy for making digital
work last. The essay argues that tools are temporary, but files can survive if
they are controlled by the user and stored in formats that remain easy to read
and retrieve.

## Core Claim

The durable object is the file, not the app. Apps, cloud services, databases,
and proprietary formats can become inaccessible, but well-chosen files preserve
the user's ability to return to their work later.

## Connection To Obsidian

Ango uses [[Obsidian]] as an example of this philosophy. The app itself is not
guaranteed to last forever, but the plain text files created through it are
designed to remain usable beyond the lifespan of the app.

## Notes

This source complements the [[LLM Wiki Pattern]] because both ideas prioritize
local, inspectable, durable markdown files over opaque application state.
