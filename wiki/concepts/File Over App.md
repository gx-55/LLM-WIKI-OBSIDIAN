# File Over App

File Over App is the principle that digital artifacts should exist as files the
user can control, retrieve, inspect, and move between tools. The file is treated
as the durable object; the app is treated as a temporary interface.

## Why It Matters

If the artifact is trapped behind a server, login, database, or proprietary
format, the user's future access depends on that tool continuing to exist. A
file-first workflow gives the user more ownership and makes long-term
preservation more plausible.

## Design Implication

Tool makers should make export, interoperability, and readable formats central
to the product rather than afterthoughts. This aligns with [[Durable Digital Artifacts]]
and with markdown-based systems such as the [[LLM Wiki Pattern]].
